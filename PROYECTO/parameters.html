Default Parameters, Rest Parameters, Named Parameters

Características

En ES5 teníamos varias maneras para manejar funciones que necesitaran valores por defecto, argumentos indefinidos y parámetros con nombre. Con ES6, podemos hacer esto y más de una forma más concisa.

Default Parameters
              

function addTwoNumbers(x, y) {
    x = x || 0;
    y = y || 0;
    return x + y;
}

Con ES6 podemos colocar valores por defecto como parámetros. 

              

function addTwoNumbers(x=0, y=0) {
    return x + y;
}

addTwoNumbers(2, 4); // 6
addTwoNumbers(2); // 2
addTwoNumbers(); // 0
REST Parameters

En ES5 manejamos los argumentos indefinidos así: 

              

function logArguments() {
    for (var i=0; i < arguments.length; i++) {
        console.log(arguments[i]);
    }
}

Usando el operador rest podemos pasar una cantidad indefinida de argumentos. 

              

function logArguments(...args) {
    for (let arg of args) {
        console.log(arg);
    }
}
Named Parameters

Uno de los patrones para manejar parámetros con nombre en ES5 era usar el patrón de objeto de opciones de JQuery. 

              

function initializeCanvas(options) {
    var height = options.height || 600;
    var width  = options.width  || 400;
    var lineStroke = options.lineStroke || 'black';
}

Podemos alcanzar los mismo utilizando la de-estructuración como parámetro. 

              

function initializeCanvas(
    { height=600, width=400, lineStroke='black'}) {
        // Use variables height, width, lineStroke here
    }

Si queremos que el valor entero sea opcional, de-estructuramos un objeto vacío. 

              

function initializeCanvas(
    { height=600, width=400, lineStroke='black'} = {}) {
        // ...
    }
