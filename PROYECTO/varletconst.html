Variables: var vs let vs const

Características

Además de var ahora tenemos acceso a dos nuevos identificadores para almacenar valores let, const. A diferencia de var, let y const no son colocados en la parte superior del alcance que los encierra. 

              

var snack = 'Meow Mix';

function getFood(food) {
if (food) {
  var snack = 'Friskies';
  return snack;
}
return snack;
}

getFood(false); // undefined

Sin embargo, cuando reemplazamos var con let, sucede lo siguiente. 

              

let snack = 'Meow Mix';

function getFood(food) {
    if (food) {
        let snack = 'Friskies';
        return snack;
    }
    return snack;
}

getFood(false); // 'Meow Mix'

Este cambio en el comportamiento muestra que es necesario ser cuidadosos cuando refactorizamos código antiguo que utilice var. Reemplazar var con let puede traer resultados inesperados.

Es importante notar que tanto let como const definen su alcance por bloques. Por eso, hacer referencia a estos identificadores antes de ser definidos, producirá un error de referencia. 

              

console.log(x); // ReferenceError: x is not defined

let x = 'hi';

Por esta razón lo que se debe hacer es dejar las variables var en el código para decir que se debe refactorizar con cuidado. Se usa let cuando las variables cambian su valor, y const cuando no lo hacen.
